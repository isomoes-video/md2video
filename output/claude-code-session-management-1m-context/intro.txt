中文标题：Claude Code 会话管理与 1M 上下文
English Title: Claude Code Session Management and 1M Context

这个视频围绕 Claude 关于 Claude Code 的一篇博客文章，讨论在 100 万 token 上下文窗口出现之后，用户应该如何更有意识地管理会话与上下文，而不是把更大的窗口误解成可以无限堆积信息的许可。视频先解释上下文窗口究竟包含哪些内容，包括系统提示词、对话历史、工具调用、工具输出以及文件读取记录，并进一步说明为什么上下文一旦变大，就会出现“上下文腐化”这样的实际问题：旧信息、无关信息和失败分支会逐步干扰当前任务。

在这个基础上，视频把 Claude Code 中每一次回复后的下一步操作，重新理解为一个上下文管理决策。它依次说明什么时候应该继续留在当前会话中，什么时候应该用 rewind 回到更早的节点，什么时候适合用 compact 压缩现有上下文，什么时候应该 clear 并重开新会话，以及什么时候应该把高噪音工作交给 subagent。视频也强调，自动压缩并不总是可靠，尤其是在会话已经长期围绕某条主线展开、而用户又突然切换到新的边缘问题时，一些原本关心的细节可能会在摘要中被丢失。最后，视频用一张决策表把这些操作方式整理成一套很实用的判断框架，核心结论是：真正重要的不是上下文窗口有多大，而是你是否在主动管理上下文的形状。
