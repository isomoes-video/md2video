中文标题：提示缓存决定智能体架构
English Title: Prompt Caching Defines Agent Architecture

这个视频基于 Anthropic 关于 Claude Code 的工程经验，讲解为什么提示缓存不是一个后期优化项，而是长时间运行智能体系统的核心架构约束。内容从缓存命中率对延迟、输入成本和速率限制的影响讲起，说明智能体之所以能够持续携带大量上下文，关键在于重复上下文必须足够便宜。

视频进一步解释提示缓存的底层机制：它本质上依赖精确的前缀匹配。因此，系统提示、工具定义、项目指令、会话上下文和实时对话的排列顺序会直接影响缓存能否复用。围绕这一点，视频总结了多个设计原则：稳定内容放在前面，变化内容放在后面；用消息承载更新，而不是重写缓存前缀；不要在会话中途切换模型或增删工具；压缩和子任务也应尽量继承父会话的稳定前缀。

最后，视频把提示缓存提升为一种运营模型：团队需要像监控可用性一样监控缓存健康，在功能设计和发布评审中关注前缀稳定性。只有从第一天起围绕缓存设计，智能体产品才能在速度、成本和可扩展性之间取得平衡。
