时间戳、工具顺序变化，或工具结构定义改动，都可能让缓存前缀失效。

为复用而组织提示

这种顺序能让许多会话共享早期缓存命中，只让实时对话部分发生变化。

智能体设计检查清单

1. 把提示缓存当作前缀约束。

2. 稳定材料放前面，实时更新放最后。

3. 保持模型和工具定义稳定。

4. 在保留父前缀的前提下分叉工作。

5. 像监控可用性一样监控缓存健康。
